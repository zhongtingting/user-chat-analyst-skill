## JSON 多媒体消息处理

部分平台的用户上传图片/文件时，`content` 字段为 JSON 数组：

```json
[
  {"type": "text", "content": "帮我分析这张图"},
  {"type": "url", "content": "https://example.com/api/files/image.png?token=..."}
]
```

处理逻辑：
1. 提取所有 `type=text` 的 content 拼接为主文本，按常规规则分类
2. 无文本时，按 URL 扩展名判断：`.png/.jpg/.gif/.webp` → 图片；`.pdf/.docx/.xlsx` → 文档；`.mp3/.wav` → 音频
3. 无法判断时归入"附件/多媒体处理"

## 卡点检测信号

卡点规则存放于 `scripts/generate_report.py` 的 `PAIN_POINT_RULES` 列表，可直接增删。

| 卡点类型 | 典型信号词 |
|---------|-----------|
| 文件下载断链 | 下载链接、给我链接、发邮箱、打包下载 |
| 安装/环境失败 | 安装失败、命令不存在、无法识别、路径错误 |
| 推送失效 | 没有收到、推送失败、未连接、不行了 |
| 任务完成无通知 | 完成了吗、是否完成、进展如何、好了吗 |
| 长任务中断 | 你还在吗、怎么停了、任务中断 |
| 安全风险 | system prompt、忽略之前、jailbreak |

## 用户画像判断逻辑

画像规则存放于 `scripts/generate_report.py` 的 `PERSONA_RULES` 列表，按优先级匹配，命中即停止。

**新增画像示例：**

```python
("医疗咨询用户",
 lambda uid, cnt, cats, txt:
     sum(1 for m in txt.split("|||")
         if any(k in m for k in ["诊断", "用药", "症状", "医院"])) >= 3),
```

添加到 `PERSONA_RULES` 列表中合适位置即可生效。
